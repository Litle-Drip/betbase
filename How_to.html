How to Read BaseScan Transaction Logs

When you create, join, vote, or payout, the contract emits events.
These events show up inside BaseScan under the “Logs” tab.
Here’s how to read them.

Event Name
Example: ChallengeOpened 
This means someone created a new challenge.

Topics Section (indexed fields)
These are fields that the blockchain makes searchable.

  For ChallengeOpened topics include:
    •	challengeId (the unique challenge number)
    •	challenger (address that created the challenge)

  Example:	 Topic 1 → challengeId = 4
    Topic 2 → challenger = 0xF7A393468e9e...
    This is where you find the challenge number used in your DApp.

Data Section (non-indexed fields)

These are all the numeric or informational values.

stakeWei: 30000000000000000    (0.03 ETH)
feeBps: 250                    (2.5% fee)
joinDeadline: 1764114909       (UNIX timestamp)
resolveDeadline: 1764116709    (UNIX timestamp)

Use epochconverter to convert timestamps, or: new Date(1764114909 * 1000)

What each field means

Field: Meaning
challengeId :Unique ID of the challenge
challenger: Address that created the challenge
stakeWei: Amount each player deposited (in wei)
feeBps: Protocol fee in basis points (250 = 2.5%)
joinDeadline:	Last time someone can join
resolveDeadline:	Last time someone can vote before refunds open
resolve window:	Time between join deadline → refund availability

Example Breakdown
  
  Event: ChallengeOpened
  Topics:
  •	Topic 1 = challengeId 4
  •	Topic 2 = challenger wallet

  Data:
  •	0.03 ETH stake
  •	2.5% fee
  •	Join deadline timestamp
  •	Resolve deadline timestamp

This means: Challenge #4 was created successfully and will follow the join → vote → payout/refund flow.

Final Notes for Users

Both players must vote.
Votes cannot be changed.
Matching votes = payout.
Mismatched votes = refund (after deadline).
Missing votes = challenge stuck forever in this version.
Use small stakes unless you trust the other player to vote.
